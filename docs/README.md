# Documentation du projet

Cette documentation couvre l’architecture, les flux de données, les workflows, l’API, les dépendances et les aspects IA de l’application.

## Arborescence des documents

- `docs/README.md` - point d’entrée et sommaire
- `docs/architecture.md` - architecture globale et structure du projet
- `docs/data-flow.md` - flux de données entre composants
- `docs/workflows.md` - parcours utilisateurs et cas d’usage principaux
- `docs/api.md` - API REST exposée par le backend
- `docs/dependencies.md` - dépendances techniques et services externes
- `docs/model-ai.md` - logique IA / machine learning, algorithmes et modèles
- `docs/data-process.md` - processus de traitement des données, ingestion, normalisation et préparation ML

## Objectif

L’objectif est de fournir une documentation claire, précise et maintenable pour une application web full stack composée d’un backend Django REST, d’un frontend React et d’un moteur de prédiction machine learning.

## Utilisation

- Ouvrir les fichiers `.md` dans le dossier `docs/`
- Utiliser la section `architecture.md` pour comprendre la structure globale
- Consulter `api.md` pour les routes disponibles
- Consulter `model-ai.md` pour les détails ML et prédictions
- Vérifier `docs/architecture.md` pour les dernières mises à jour Docker et réseau frontend/backend
